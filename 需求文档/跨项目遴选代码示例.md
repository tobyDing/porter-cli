Git 可以跨项目遴选（cherry-pick）提交，但需要一些额外步骤，因为 cherry-pick 默认只作用于当前仓库内的提交。要将一个项目的提交应用到另一个项目，可以按以下方式操作：

---

### ✅ 方法：添加远程仓库作为源（推荐）

假设：

- **项目 A**：源项目（包含你想遴选的提交）
- **项目 B**：目标项目（你想把提交应用到这里）

#### 步骤如下：

1. **进入项目 B 的本地仓库目录**

   ```bash
   cd project-b
   ```

2. **添加项目 A 作为远程仓库（临时）**

   ```bash
   git remote add project-a /path/to/project-a
   # 或者用 HTTPS/SSH（如果项目 A 在远程）：
   # git url 例子：git@github.com:user/project-a.git
   # git remote add project-a https://github.com/user/project-a.git
   ```

3. **获取项目 A 的提交历史**

   ```bash
   git fetch project-a
   ```

4. **使用 cherry-pick 应用指定提交**

   ```bash
   git cherry-pick <commit-hash-from-project-a>
   ```

   > 如果提交中涉及的文件路径在项目 B 中不存在，可能会冲突，需手动解决。

5. **清理（可选）**
   ```bash
   git remote remove project-a
   ```
